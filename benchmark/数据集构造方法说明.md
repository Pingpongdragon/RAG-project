# 实验数据集构造方法说明

## 1. 问题背景

在 RAG 系统中，知识库（KB）需要随时间更新以适应用户兴趣的变化。为了评估不同 KB 更新策略的效果，我们需要构造**带有可控 topic 漂移模式的查询流**，模拟用户兴趣随时间的变化。

## 2. 核心思路

我们将实验数据集定义为两部分：
- **查询流**（query stream）：$N$ 条按时间排列的查询，每条查询属于某个自然 topic
- **文档池**（document pool）：$M$ 篇候选文档，KB 更新策略从中选择子集存入 KB

关键在于**查询流的 topic 分布随时间变化**——通过不同的概率调度函数 $P(\text{topic} \mid t)$ 来控制，其中 $t = \frac{i}{N-1} \in [0, 1]$ 是查询在流中的归一化位置。

每一步的采样过程：

$$\text{topic}_i \sim P(\cdot \mid t), \quad t = \frac{i}{N-1}$$

即在第 $i$ 步，先由调度器计算当前各 topic 的概率，再按该分布随机采样一个 topic，从该 topic 的查询池中取出下一条查询。

## 3. 三种 Topic 调度器

### 3.1 Gaussian Drift（渐进漂移）

$K$ 个 topic 的中心 $c_k$ 等间距分布在 $[0, 1]$ 上，每个 topic 的激活权重为高斯函数：

$$w_k(t) = \exp\left(-\frac{(t - c_k)^2}{2\sigma^2}\right)$$

归一化后得到概率分布：

$$P(\text{topic}_k \mid t) = \frac{w_k(t)}{\sum_{j=1}^{K} w_j(t)}$$

$\sigma$ 控制相邻 topic 的重叠程度：$\sigma$ 越大，漂移越平缓；$\sigma$ 越小，topic 切换越集中。

**实际效果示例**（3 个 topic, $\sigma=0.18$）：

| 时刻 $t$ | Pasta | Ferrari | Jazz | 效果 |
|-----------|-------|---------|------|------|
| 0.00 | 0.98 | 0.02 | 0.00 | 几乎只采样 Pasta |
| 0.25 | 0.50 | 0.50 | 0.00 | 过渡区：两 topic 共存 |
| 0.50 | 0.02 | 0.96 | 0.02 | 切换到 Ferrari 为主 |
| 0.75 | 0.00 | 0.50 | 0.50 | Ferrari→Jazz 过渡 |
| 1.00 | 0.00 | 0.02 | 0.98 | 几乎只采样 Jazz |

### 3.2 Sigmoid Shift（突变转移）

每对相邻 topic 之间设置一个 sigmoid 过渡点 $t_{k}$。Topic $k$ 的权重是左侧"开启"和右侧"关闭"两个 sigmoid 的乘积：

$$w_k(t) = \sigma_L^{(k)}(t) \cdot \sigma_R^{(k)}(t)$$

其中 $\sigma_L^{(k)}(t) = \frac{1}{1 + e^{-s(t - t_{k-1})}}$，$\sigma_R^{(k)}(t) = \frac{1}{1 + e^{s(t - t_{k})}}$，$s$ 是陡峭程度参数。

当 $s=30$ 时，切换在 2-3 步内完成，模拟用户兴趣的**突然改变**。

**实际效果**：
| 时刻 $t$ | 主导 topic | 概率 |
|-----------|-----------|------|
| 0.00 ~ 0.15 | Pasta | > 0.88 |
| 0.17（过渡点） | Ferrari | 0.53 |
| 0.20 ~ 0.48 | Ferrari | > 0.73 |
| 0.50（过渡点） | Jazz | 0.50 |
| 0.60 ~ 1.00 | Jazz | > 0.95 |

### 3.3 Cyclic Schedule（周期回归）

每个 topic 在 $[0, 1]$ 上拥有 $C$ 个等间距高斯峰：

$$w_k(t) = \sum_{j=0}^{C-1} \exp\left(-\frac{(t - p_{k,j})^2}{2\sigma^2}\right)$$

其中 $p_{k,j}$ 是第 $k$ 个 topic 在第 $j$ 个周期中的峰值位置。

**实际效果**（3 个 topic, 2 个周期）：

$$\text{Pasta} \rightarrow \text{Ferrari} \rightarrow \text{Jazz} \rightarrow \text{Pasta} \rightarrow \text{Ferrari} \rightarrow \text{Jazz}$$

这测试 KB 策略在旧 topic 重新出现时能否快速恢复——如果策略删除了旧 topic 的文档，在 topic 回归时检索质量会下降。

## 4. 数据来源

### Wizard of Wikipedia（WoW）

WoW 是微软开放域知识对话数据集，每段对话标注了自然 topic 标签（如 "Pasta", "Ferrari", "Jazz"）。我们利用这些天然标签：
- 每个 turn 的用户发言 → `QueryItem.question`
- Wizard 的回复 → `QueryItem.answer`
- 被选中的知识段落 → `gold_doc_ids`
- 所有候选知识段落 → `PoolDocument`

WoW validation 包含 1948 段对话，每个 topic 下有约 200~300 条查询和 4000~5000 篇候选文档。

### HotpotQA（实体图游走）

HotpotQA 是多跳问答数据集，每个问题的 supporting facts 来自多篇 Wikipedia 文章。我们利用共享 supporting fact title 构建实体邻接图，通过**贪心游走**产生无需分类的自然 topic 漂移：共享实体的相关问题聚在一起，逐步过渡到远处的话题。

## 5. 文档池构造

文档池的作用是提供**检索环境**，模拟 KB 更新策略面对的真实场景。池中文档包括：

| 类型 | 说明 | 占比 |
|------|------|------|
| Gold 文档 | 当前查询的正确答案来源 | ~5% |
| 同 topic 非 gold 文档 | 合理但不是最佳的干扰项 | ~30-40% |
| 其他 topic 文档 | 完全无关的噪声 | ~55-65% |

`merge_pool()` 确保：
1. 所有 gold 文档必定存在于池中（否则实验无意义）
2. 其余位置随机采样到上限（默认 5000），保持多样性
3. 去重后打乱顺序

## 6. 完整的数据集构建流程

```
     WoW JSON
        │
        ▼
   load_wow()                    ← 加载原始数据 (1948 段对话)
        │
        ▼
   select_topics()               ← 选择数据充足的 topics
        │
        ▼
   extract_topic_data()          ← 按 topic 提取 queries 和 docs
        │
        ├──→ topic_queries: {topic: [QueryItem, ...]}
        └──→ topic_docs:    {topic: [PoolDocument, ...]}
                │
    ┌───────────┤
    │           ▼
    │    TopicSchedule.get_probs(t)
    │           │
    │           ▼
    │    build_stream()          ← 按 P(topic|t) 逐步采样 → query_stream
    │
    ▼
   merge_pool()                  ← gold 文档优先 + 随机填充 → document_pool
        │
        ▼
   ExperimentDataset             ← 最终输出
    .query_stream   (300 条查询)
    .document_pool  (5000 篇文档)
```

## 7. 四种实验场景

| 实验 | 数据源 | 调度器 | 查询数 | 池大小 | 测试目标 |
|------|--------|--------|--------|--------|----------|
| Gradual Drift | WoW | Gaussian ($\sigma$=0.18) | 300 | 5000 | 对缓慢漂移的跟踪能力 |
| Sudden Shift | WoW | Sigmoid ($s$=30) | 300 | 5000 | 对突变的反应速度 |
| Cyclic Return | WoW | Cyclic (2 cycles) | 300 | 5000 | 对旧 topic 回归的恢复能力 |
| HotpotQA Walk | HotpotQA | 实体图游走 | 400 | 50000 | 多跳推理 + 自然漂移 |

## 8. 快速使用

```python
from benchmarks import build_gradual_drift

# 构建一个渐进漂移数据集
ds = build_gradual_drift(total_queries=300, sigma=0.18)

print(len(ds.query_stream))    # 300
print(len(ds.document_pool))   # ~5000
print(ds.query_stream[0].question)   # 第一条查询
print(ds.query_stream[0].gold_doc_ids)  # 对应的 gold 文档 ID
```
