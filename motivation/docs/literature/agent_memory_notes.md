在多用户共享记忆的叙述层面，目前的 Agent 领域对“共享”的理解非常局限。综述指出，现有的多智能体共享记忆主要停留在项目协作层面，例如几个角色 Agent 共享同一个对话上下文或代码草稿。我们将这一概念升维到了“系统级多用户共享知识基础设施”。Shared KB 不再是简单的共享文件夹，而是一个容量受限、需要动态调度的公共记忆资源池。这就自然引出了知识库热点管理和淘汰置换的工程挑战。

我们在理论定位上发现的最核心的 Gap，是现有的 KB 更新机制对“用户需求分布（Query Distribution）”是完全盲目的。综述中提到的所有最前沿的记忆更新策略，无论是由规则触发还是由强化学习控制，全都是基于单个 Agent 的微观交互事件来触发的。整个学术界都缺乏一种宏观的、基于群体查询分布偏移（Topic Shift/Concept Drift）来主动重组知识库物理布局的机制。
