# KB 更新策略实验对比报告

> 实验时间: 2026-03-19 | Embedding: nomic-embed-text-v1.5 | KB Budget=50 | 300 queries × 3 场景

---

## 1. 三种方法的核心算法对比

### 1.1 QARC（我们的方法）— Query-Alignment Reactive Curation

**核心思想**: 主动检测用户兴趣漂移，按需重选 KB 文档组合。

**算法流程**:
1. **兴趣建模**: 每个窗口（20 queries）用 AutoKMeans 聚类 → 得到兴趣中心 $\{c_k\}$ 和权重 $\{w_k\}$
2. **漂移检测 (Part 1)**: DriftLens — 计算 query-KB 对齐特征的 FID 分数，与基线对比检测分布漂移
   - 对齐特征 = [sim(q, c₁), ..., sim(q, cₖ), top1_sim, ..., topN_sim]
   - FID(当前窗口, 基线) > threshold → 漂移发生
3. **决策 (Part 2)**: KB Agent — 根据漂移程度和 AlignmentGap 选择更新策略
   - AlignmentGap = 1 - avg(max_sim(q, kb_docs))：衡量 KB 与当前兴趣的偏离程度
   - 无漂移 → NO_OP
   - 漂移 + gap 小 → 温和更新 (λ=0.1, 少量替换)
   - 漂移 + gap 大 → 激进更新 (λ=0.5, 大幅替换)
4. **执行更新**: 按兴趣权重从 Pool 中重选文档
   - 每个兴趣簇 $k$: 取 Pool 中与 $c_k$ 最近的 top 文档，按 $w_k$ 分配名额
   - 替换比例由 λ_max 控制

**关键优势**: 文档级 KB 重组 — 直接换入与当前兴趣匹配的文档，换出不相关的。

### 1.2 ComRAG — Conversational RAG with QA Memory

**核心思想**: KB 完全不动，靠积累 QA 历史记忆来弥补 KB 的不足。

**算法流程**:
1. **KB 固定**: 初始化后 KB 不再变化（等同于 StaticKB）
2. **QA 记忆库**: 双向量库 V_high（高质量 QA）+ V_low（低质量 QA），基于质心聚类管理
3. **三层路由** (每条 query):
   - sim ≥ δ(0.9) → **直接复用**: 从 V_high 中找到几乎一样的历史问题，直接返回历史答案
   - τ(0.75) ≤ sim < δ → **参考生成**: 用高质量 QA 作为 ICL 示例辅助 LLM
   - sim < τ → **KB 回退**: 标准 KB 检索 + V_low 中的低质量回答作为反面教材
4. **记忆更新**: 每次 QA 后自动存入高质量或低质量库

**关键特点**: 不改 KB 内容，只做查询复用/路由。适合对话式场景中重复模式多的情况。

### 1.3 ERASE — Editable RAG with Self-consistent Editing

**核心思想**: 用新到的 gold 文档"编辑"KB 中已有的事实条目。

**算法流程**:
1. **KB 初始化**: 将文档存为"原子事实"条目，每条附带真假历史记录
2. **query 到来时**: 从 KB 检索 top-k 事实
3. **gold 文档到来后** (事后反馈):
   - 计算 gold 文档与 KB 中最相似条目的 sim
   - sim ≥ threshold(0.7) → **REWRITE**: 用新内容覆写最相似的旧条目（知识更新）
   - sim < threshold → **ADD**: 作为新事实加入 KB（超出 budget 则淘汰最旧的）
4. **去假存真**: 每条事实保留时间线，检索时可将真假信息一并提供给 LLM

**关键特点**: 文档驱动、事后更新。依赖 gold 文档反馈，不主动感知兴趣变化。

## 2. 核心差异总结

| 维度 | QARC (ours) | ComRAG | ERASE |
|------|-------------|--------|-------|
| **KB 是否变化** | ✅ 主动重选文档组合 | ❌ 完全不动 | ⚠️ 逐条编辑/替换 |
| **感知兴趣变化** | ✅ 显式检测 (FID + AlignmentGap) | ❌ 隐式 (sim 阈值路由) | ❌ 纯被动 |
| **更新粒度** | 文档级 — 整批替换 | 无更新 — 只做路由 | 事实级 — 逐条改写 |
| **更新触发** | 检测到漂移时 | 从不更新 KB | 每次收到 gold doc |
| **适应速度** | 快 (1个窗口内完成) | 慢 (依赖历史积累) | 中 (逐条累积) |
| **依赖** | 查询流 + Pool | 查询流 + 历史 QA | 查询流 + Gold 反馈 |

## 3. 实验结果

### 3.1 实验设置

| 参数 | 值 |

| KB 容量 | 50 篇文档 |

### 3.3 关键观察

**QARC 全面领先**:
- **Recall@k**: QARC 平均 0.376，ERASE 0.199 (+89%)，ComRAG 0.088 (+328%)
- **Gold-in-KB**: QARC 平均 43.2%，ERASE 26.7% (+62%)，ComRAG 0.3% (KB 不更新)
- **Topic Alignment**: QARC 平均 0.710，ERASE 0.655，ComRAG 0.546

**为什么 QARC 有效？**
1. **主动感知**: DriftLens 检测到兴趣变化后立即行动，而非被动等待
2. **文档级重组**: 一次性批量替换多篇文档，比 ERASE 的逐条编辑更高效
3. **对齐驱动**: AlignmentGap 精确量化 KB 与兴趣的偏离度，指导更新力度

**为什么 ComRAG 表现不佳？**
- ComRAG 的 KB 完全不变（等同于 StaticKB），只靠 QA 记忆路由
- 在 topic 漂移场景中，KB 中缺少新 topic 的文档，记忆库也无法弥补
- 适合重复查询多的对话场景，不适合兴趣快速变化的场景

**为什么 ERASE 是第二名？**
- ERASE 能通过 gold doc 反馈逐步更新 KB 内容
- 但逐条编辑速度慢，一个窗口内只能改几条，跟不上 QARC 的整批替换
- 在 Sudden Shift 场景中差距最大（漂移速度快，逐条跟不上）

## 4. 指标说明

| 指标 | 含义 | 越高越好？ |
|------|------|:---:|
| **Recall@k** | 检索的 top-k 中包含 gold doc 的查询比例 | ✅ |
| **Gold-in-KB** | gold doc 存在于当前 KB 中的查询比例 (KB质量上界) | ✅ |
| **Topic Alignment** | KB topic 分布与当前查询 topic 分布的余弦相似度 | ✅ |
| **Adapt Speed** | 漂移后恢复对齐的窗口数 (越小越快) | ❌ 越小越好 |
